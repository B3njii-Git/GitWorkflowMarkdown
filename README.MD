# Proyecto de Automatización y Gestión con Git

[cite_start]Este repositorio ha sido creado para demostrar la implementación de un entorno de trabajo colaborativo utilizando **Git** y la automatización de calidad de código mediante **GitHub Actions**[cite: 49].

## 📋 Descripción del Proyecto

[cite_start]El objetivo principal es configurar un flujo de trabajo profesional que asegure que toda la documentación en formato Markdown cumpla con estándares de calidad antes de ser integrada en las ramas principales[cite: 49, 68].

### Características Principales:
* [cite_start]**Modelo de Ramas:** Uso de GitFlow/TBD con ramas `main` y `develop`[cite: 53, 54].
* [cite_start]**Integración Continua:** Linter automático para archivos `.md`[cite: 66, 68].
* [cite_start]**Estándares de Documentación:** Cumplimiento de reglas específicas mediante `.markdownlint.json`[cite: 69].
